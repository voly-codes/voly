# Spend-протокол (v1)

HTTP-интерфейс сервиса учёта расходов. Ядро говорит с ним через
`voly/spend/client.py:SpendClient`; реализация сервера может быть любой —
референсный однопользовательский воркер лежит в `cf-workers/` (Durable
Objects), self-hosted и командные (мультитенантные) реализации подключаются
тем же протоколом через `CF_WORKER_SPEND_URL` / `spend.remote_url`.

Это **версионируемый публичный контракт** ядра: формы запросов/ответов
заморожены контрактными тестами (`tests/test_protocol_contracts.py`).
Изменение = бамп версии здесь + обновление снимка в тесте.

## Аутентификация

`Authorization: Bearer <token>` — токен из `CF_WORKER_SPEND_TOKEN`
(fallback `CLOUDFLARE_API_TOKEN`). Все запросы/ответы — `application/json`.

## Endpoints

### `GET /health`

Проверка доступности. Ответ: `{"status": "ok"}` (форма не фиксируется,
важен HTTP 200).

### `POST /spend/record`

Записать расход по задаче. Тело (все поля обязательны, пустые строки
допустимы):

```json
{
  "agent": "developer",
  "cost_usd": 0.25,
  "task_id": "uuid",
  "model": "claude-sonnet-4-6",
  "provider": "anthropic"
}
```

Ответ: HTTP 200, тело игнорируется.

### `GET /spend/check?agent=<agent>&limit=<daily_limit>`

Проверить, не превышен ли дневной лимит агента. Ответ:

```json
{"ok": true, "spent": 1.23, "limit": 20.0}
```

`ok=false` → вызывающая сторона останавливает задачу со статусом
`spend_limited` (см. `pipeline/stages.py:_stage_spend_check`).

### `GET /spend/summary?days=<n>`

Агрегат расходов за n дней. Форма ответа — по усмотрению сервера
(референс: суммы по агентам/дням); ядро передаёт его в UI как есть.

### `GET /spend/recent?limit=<n>`

Последние записи: `{"entries": [ { ... }, ... ]}` — ядро читает только
ключ `entries`.

## Границы v1

- Идентификация организации/пользователя вне протокола (токен = арендатор);
  мультитенантные реализации решают это на своей стороне.
- Валюта — USD, поле `cost_usd`.
- Ретраи/идемпотентность `record` не специфицированы (события телеметрии —
  источник истины для сверки; см. `TaskEvent` в `docs/backend/api.md`).
