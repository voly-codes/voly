> **Что это:** [`codebase-memory-mcp`](https://github.com/DeusData/codebase-memory-mcp)

> Локальный репо: `/home/lanies/git/codeops/codebase-memory-mcp`.

> **Статус:** референс / кандидат в бэклог. **Кода не переносим** — C-движок под

## Зачем смотрели

Оценка перед задачей **этапа 1 «ключ кэша executor path» (риск R1)**: нужен
дешёвый снимок состояния проекта в ключе кэша, иначе код-задачи отдают протухший
результат.

---

## Что там прямо релевантно (техника, не код)

| Компонент репо | Что делает | Чем полезно VOLY |
|---|---|---|
| `src/git/git_context.{c,h}` | Резолвит `head_sha`, `base_sha`, `branch`, `worktree_root`, флаги `is_git/is_worktree/is_detached` | Готовая форма **«снимок состояния репо»** для ключа кэша |
| `src/pipeline/pipeline_incremental.c` | Детект изменений: **mtime+size против сохранённых хэшей → changed/unchanged**, реиндекс только изменённого | Стратегия дешёвой инвалидации кэша executor path |
| `src/simhash/minhash.c` | similarity-хэш содержимого | На потом — near-duplicate/семантика кэша |
| `src/watcher/watcher.c`, `src/pipeline/pass_gitdiff.c` | Файловый watcher, git-diff pass | Референс для live-инвалидации |

## Вывод для задачи cache-key R1: код НЕ берём

Техника подтверждена рабочим движком, но она тривиальна (порядка 15 строк на
Python). Переносить нечего — ценность репо здесь в **валидации дизайна**:

> **Ключ кэша executor path** = `git rev-parse HEAD` (+ dirty-флаг рабочего
> дерева) **и** хэш(`mtime`+`size`+`content`) затронутых задачей файлов.
> Именно так классифицирует changed/unchanged их `pipeline_incremental`.

Границы валидности кэша документируем в `docs/backend/ai-gateway.md` /
`docs/backend/executors.md`: кэш код-задачи валиден только для того же
git-снимка + неизменённого набора релевантных файлов.

Идём **своим путём** (git HEAD + file-hash), `pipeline_incremental` — как
референс стратегии инвалидации.

## Отдельная возможность (бэклог, НЕ сейчас)

MCP этого движка — качественная замена шага VOLY **«local context: собрать
релевантные файлы через grep»** перед executor (knowledge-graph query вместо
десятков grep/read; заявлено 120× меньше токенов). VOLY уже умеет MCP
(`voly mcp`), поэтому подключается как **внешний инструмент, не форк**.

Почему не сейчас (`CLAUDE.md`, roadmap §7/§10): **не наращивать периферию до
стабилизации ядра B** и не делать без явного запроса. Зафиксировано в чеклисте
как кандидат этапа 7.
