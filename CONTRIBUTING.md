# Contributing to VOLY

Спасибо за интерес к VOLY! Ядро проекта открыто под [Apache 2.0](LICENSE),
и вклад в него приветствуется.

## Как внести вклад

1. Форкните репозиторий и создайте ветку от `main`.
2. Внесите изменения. Правила проекта — в [CLAUDE.md](CLAUDE.md):
   - **Docs move with code** — изменение поведения кода сопровождается
     обновлением соответствующего файла в `docs/` в том же коммите.
   - **Gateway first** — вызовы моделей идут через `AIGateway.chat()`
     (кроме executor-ов).
   - **Project-agnostic core** — никакой продуктовой логики и hardcoded
     путей в `voly/`.
3. Прогоните тесты и гейты:
   ```bash
   python3 -m pytest tests/ -q
   python3 scripts/check_doc_links.py
   python3 scripts/check_env_doc_sync.py
   ```
   Новые тесты — mock-based, без обращений к реальным API.
4. Откройте pull request с описанием мотивации изменения.

## Developer Certificate of Origin (DCO)

Проект использует [DCO](https://developercertificate.org/). Подписывайте
каждый коммит:

```bash
git commit -s -m "..."
```

Строка `Signed-off-by: Имя <email>` подтверждает, что у вас есть право
передать этот код проекту под его лицензией.

## Границы open-core

VOLY развивается по модели open-core. Чтобы ожидания были честными с обеих
сторон, граница публична:

**Открытое ядро (этот репозиторий):** оркестрация, executor chain, billing
fallback chain, мульти-агентная декомпозиция, AI Gateway (кэш, лимиты,
fallback), телеметрия, CLI, web UI для одного пользователя. PR, улучшающие
ядро, — всегда добро пожаловать.

**Коммерческая обвязка (вне этого репозитория):** командный hosted control
plane — организационные дашборды расходов, org-level spend limits, SSO,
аудит-лог, managed-хостинг federation. PR, реализующие такие функции в ядре,
мы, скорее всего, вежливо отклоним с объяснением — не потому что вклад
плох, а чтобы граница оставалась предсказуемой.

Пограничные случаи обсуждаются в issue до написания кода — это сэкономит
время и вам, и мейнтейнерам. Протоколы, по которым ядро общается с любыми
внешними сервисами (телеметрия `TaskEvent`, spend-протокол, A2A), открыты
и версионируются — self-hosted альтернативы приветствуются.

## Название

«VOLY» — название проекта. Форки кода разрешены лицензией; распространение
форков под тем же названием — нет.
